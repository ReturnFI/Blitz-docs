    توضیحات صفحه تنظیمات

    راهنمای صفحه تنظیمات

    این داکیومنت یک راهنمای جامع برای همه گزینه‌های موجود در صفحه تنظیمات ارائه می‌دهد. صفحه تنظیمات از چند تب تشکیل شده که هرکدام بخشی خاص از سرویس را مدیریت می‌کنند.

    اشتراک‌ها (Subscriptions)

    این تب مربوط به وب‌سروری است که لینک‌های اشتراک کاربران را ارائه می‌دهد.

    کنترل سرویس

    این بخش کنترل اصلی سرویس اشتراک است.

    • Domain: دامنه‌ای که سرویس اشتراک روی آن در دسترس خواهد بود (مثلاً sub.yourdomain.com). توجه کنید که از http:// یا https:// استفاده نکنید.
    • Port: پورتی که سرویس روی آن گوش می‌دهد (مثلاً 8080).
    • Start Service: بعد از وارد کردن دامنه و پورت، روی این دکمه بزنید تا وب‌سرور شروع به کار کند.
    • Stop Service: اگر سرویس در حال اجرا باشد، این دکمه ظاهر می‌شود و می‌توانید آن را متوقف کنید.

    پیکربندی لینک

    این زیربخش فقط زمانی فعال می‌شود که سرویس اشتراک شروع به کار کرده باشد. به شما امکان می‌دهد مسیر لینک‌های اشتراک را شخصی‌سازی کنید.

    • Subscription Path Segment: یک رشته ساده و منحصربه‌فرد (مثل my-sub) وارد کنید تا مسیر اختصاصی بسازید. این کار باعث می‌شود لینک‌ها کمتر قابل پیش‌بینی باشند. مثلاً:
      http://sub.yourdomain.com:8080/my-sub/...
    • Save Subpath: برای اعمال مسیر جدید کلیک کنید.

    ربات تلگرام (Telegram Bot)

    این تب برای اتصال و مدیریت یک ربات تلگرام جهت کارهای مدیریتی و دریافت نوتیفیکیشن‌ها است.

    • API Token: توکن اختصاصی ربات تلگرام (از BotFather دریافت می‌شود).
    • Admin ID: شناسه عددی تلگرام شما برای دسترسی مدیریتی به ربات.
    • Automatic Backup Interval (Hours): فاصله زمانی (ساعت) برای بکاپ خودکار (مثلاً 12 برای هر ۱۲ ساعت).
    • Start: بعد از وارد کردن توکن و Admin ID، این دکمه ربات را فعال می‌کند.
    • Stop: برای متوقف کردن ربات.
    • Save Interval: در حالتی که ربات فعال است، می‌توانید بازه زمانی بکاپ را تغییر دهید و بدون ری‌استارت ذخیره کنید.

    تنظیمات Hysteria

    این بخش شامل تنظیمات اصلی سرویس Hysteria است.

    • Change Port: تغییر پورت اصلی سرویس Hysteria برای اتصال کلاینت‌ها.
    • Change SNI: تنظیم یا تغییر دامنه SNI برای شبیه‌سازی ترافیک وب‌سایت واقعی و جلوگیری از شناسایی.

    دقت داشته باشید با تغییر SNI مقادیر pinSHA256 در فایل config.json تغییر خواهد کرد و در صورت قطع شدن کاربران باید دوباره کانفیگ دریافت کنن.

    • OBFS: فعال/غیرفعال کردن مخفی‌سازی ترافیک. وضعیت فعلی نمایش داده می‌شود و می‌توانید آن را تغییر دهید.

    دقت داشته باشید با تغییر OBFS مقادیر OBFS در فایل config.json تغییر خواهد کرد و در صورت قطع شدن کاربران باید دوباره کانفیگ دریافت کنن.

    • Update GeoIP & GeoSite Files: دانلود آخرین دیتابیس GeoIP و GeoSite برای کشورها (ایران، چین، روسیه). این فایل‌ها برای مسیریابی درست ضروری هستند.

    مدیریت IP (IP Management)

    این تب برای مدیریت آدرس‌های IP و سرورهای خارجی است.

    آدرس/دامنه سرور اصلی

    • IPv4 / Domain: آدرس IPv4 یا دامنه عمومی سرور.
    • IPv6 / Domain: آدرس IPv6 یا دامنه عمومی سرور.

    نودهای خارجی

    در صورت داشتن چند سرور، می‌توانید نودهای خارجی را اضافه کنید.

    • Add New Node:
    • وارد کردن نام نود (مثلاً Node-US).
    • وارد کردن IP یا دامنه نود.
    • کلیک روی Add Node.
    • Manage Nodes: لیست نودها نمایش داده می‌شود و امکان حذف هر نود وجود دارد.

    کانفیگ‌های اضافی (Extra Configs)

    با این قابلیت می‌توانید لینک‌های پروکسی خارجی (مثل Vmess، Vless، SS یا Trojan) را به صورت خودکار به لینک‌های اشتراک همه کاربران اضافه کنید.

    • Add New Configuration:
    • وارد کردن نام یکتا (مثلاً My-Vmess-Link).
    • چسباندن URI کامل (مثلاً vmess://...).
    • کلیک روی Add Config.
    • Manage Configurations: جدول کانفیگ‌ها نمایش داده می‌شود و امکان حذف هرکدام وجود دارد.

    بکاپ (Backup)

    این بخش برای پشتیبان‌گیری و بازیابی کل پنل و تنظیمات Hysteria است.

    • Restore from Backup:
    • انتخاب فایل .zip بکاپ.
    • کلیک روی Upload and Restore → تأیید → سیستم به حالت بکاپ بازگردانده می‌شود.
    • Create New Backup:
    • کلیک روی Download Backup → دانلود یک فایل .zip شامل تمام تنظیمات فعلی.

    محدودیت IP (IP Limit)

    این تب برای محدود کردن تعداد IPهایی است که یک کاربر می‌تواند همزمان استفاده کند.

    کنترل سرویس

    • Start: فعال‌سازی سرویس محدودیت IP.
    • Stop: غیرفعال کردن سرویس.

    پیکربندی

    (فقط وقتی سرویس فعال باشد در دسترس است)

    • Block Duration (seconds): مدت زمان بلاک شدن IP بعد از عبور از محدودیت.
    • Max IPs per User: حداکثر تعداد IP مجاز برای هر کاربر.
    • Save Configuration: ذخیره تغییرات.

    سایت جعلی (Decoy Site)

    برای ایجاد یک سایت ظاهری استفاده می‌شود تا سرویس پروکسی کمتر شناسایی شود.

    • Domain: دامنه‌ای که به سایت جعلی اشاره کند.
    • Decoy Site Path: مسیر مطلق روی سرور که فایل‌های سایت (مثل index.html) قرار دارند.
    • Setup Decoy: فعال‌سازی سایت جعلی.
    • Stop Decoy: توقف سایت جعلی.
    • Status: وضعیت فعلی نمایش داده می‌شود (فعال یا غیرفعال).

    WARP

    مدیریت سرویس Cloudflare WARP برای مسیریابی ترافیک خروجی سرور.

    • Install & Start WARP: نصب و فعال‌سازی WARP (اگر فعال نباشد).
    • Stop & Uninstall WARP: توقف و حذف کامل WARP (اگر فعال باشد).

    پیکربندی مسیر‌یابی WARP

    وقتی WARP فعال باشد:

    • Route All Traffic through WARP: تمام ترافیک خروجی از WARP عبور کند.
    • Route Popular Sites through WARP: فقط سایت‌های بین‌المللی محبوب از WARP عبور کنند.
    • Route Domestic Sites through WARP: فقط سایت‌های داخلی از WARP عبور کنند.
    • Block Adult Sites (WARP Family DNS): فعال‌سازی فیلتر DNS برای مسدود کردن محتوای بزرگسال.
    • Save Configuration: ذخیره تنظیمات مسیریابی.
